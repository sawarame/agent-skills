# 仕様生成スキル

このスキルは、現在のプロジェクトのコミット履歴を分析し、プロジェクトに適した仕様をマークダウン形式で自動生成するためのものです。エージェントは以下の手順で動作します。

## 手順
  1. 先ず仕様書の場所を特定します（`SPEC.md`や`docs/spec.md`など）。場所が不明な場合はユーザーに確認してください。
  2. 仕様書が存在しない場合は、新規に作成するかどうかユーザーに確認してください。
  3. 仕様書がしない場合は、プロジェクトのコミット履歴を分析して、プロジェクトの目的、機能、使用方法などを含む仕様をマークダウン形式で生成してください。
  4. すでに仕様書が存在する場合は下記の手順で差分を追跡し、必要に応じて更新してください。
     - `git log -1 <仕様書のパス>`　を実行して、仕様書が最後に更新されたコミットを特定します。
     - `git log <最後の更新コミット>..HEAD` を実行して、仕様書の最後の更新以降のコミット履歴を取得します。
     - コミット履歴を分析して、仕様書の内容と照らし合わせます。
     - 仕様書に記載されていない新しい機能や変更点があれば、それらを仕様書に追加する提案をユーザーに行います。
     - 既存の内容と矛盾する変更があれば、ユーザーに確認して修正案を提示してください。

## 厳守事項
- `git log` 以外のコマンドはこのスキルの範囲外です。仕様書の内容を変更する際は、ユーザーに提案し、ユーザーが手動で変更を加えることを推奨してください。
- `git add` や `git commit` などのコマンドはこのスキルの範囲外です。仕様書の内容を変更する際は、ユーザーに提案し、ユーザーが手動で変更を加えることを推奨してください。
