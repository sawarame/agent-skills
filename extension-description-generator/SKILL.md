# Store Description Generator

## 概要
このスキルは、高品質でプロフェッショナルな Chrome ウェブストア用説明文の自動生成を可能にします。英語と日本語の両方で一貫性を保ちながら、主要な機能、技術的な利点、およびユーザーのメリットを強調します。

## ガイドライン

### 1. 情報の収集
説明文を生成する前に、必ず以下のファイルを分析してください：
- `manifest.json`: 拡張機能名、バージョン、権限、オムニボックスのキーワードを確認します。
- `package.json`: プロジェクト名、説明、依存関係を確認します。
- `src/` または `js/` ディレクトリ: 実際の挙動や機能を理解します。
- 既存の `store_description.md`: トーン、スタイル、既存のフォーマットを維持するために参照します。

### 2. コンテンツ構造
説明文は、英語と日本語の両方で以下の標準的な構造に従う必要があります：

- **名前**: 拡張機能の正式名称。
- **短い説明**: 簡潔で印象的な1文の要約（Chrome ウェブストアでは最大132文字）。
- **詳細説明**:
    - **目的**: 拡張機能の目的の簡潔な概要。
    - **主な機能**: 主要な機能の箇条書きリスト。
    - **特徴的な機能**: 他の似たようなアプリや機能との差別化ポイント。
    - **使用方法**: ステップバイステップの使用方法。
    - **開発者/上級者向け機能**: 技術的なメリットがあれば記載。
    - **プライバシー最優先**: データの保存とプライバシーに関する声明。

### 3. 言語要件
- **英語**: プロフェッショナルで魅力的な、分かりやすいアメリカ英語を使用します。
- **日本語**: 開発者や技術に詳しいユーザーに適した、自然で丁寧（です・ます調）かつ現代的な日本語を使用します。用語が一般的な日本の技術用語と一致していることを確認してください。
- **スタイル制限**:
    - **Markdown非対応（ストア転記用本文）**: `store_description.md` ファイル内の `Name`, `Short Description`, `Detailed Description` などの大見出し（ストアの入力欄に対応する部分）は `### **...**` として残します。しかし、**`Detailed Description` の中身（本文）には一切 Markdown を使用しないでください**。本文内の小見出し（コマンド一覧など）もプレーンテキストにしてください。
    - **視認性のための記号装飾**: プレーンテキスト内での視認性を高めるため、見出しや強調には記号を効果的に使用してください。
        - **英語版**: 全角記号は避け、半角記号（`[ ]`, `-`, `*` など）を使用します（例: `[ CORE FEATURES ]`, `- Feature`）。
        - **日本語版**: 全角記号（`【 】`, `■`, `・` など）を使用します（例: `【 主な機能 】`, `■ 機能`）。
    - **絵文字は最小限に**: 絵文字を多用せず、各セクションの見出し（ 💡 など）程度に留めてください。
    - **「AI的」な表現の禁止**: 「〜してみましょう！」「〜するだけです！」といった、AIが生成したような過度に親感的、あるいは煽り気味な表現は避けてください。
    - **安易な誇張表現の禁止**: 「爆速」や「爆誕」といった、安易で使い古された誇張表現は避けてください。
    - **客観的・事実ベース**: 主観的な表現（例：「最高に便利」「素晴らしい」など）やユーザーへの問いかけは避け、機能やメリットを客観的事実に基づいて簡潔に記述してください。

## ワークフロー

1.  **メタデータの分析**: `manifest.json` と `package.json` から名前、バージョン、主要なキーワードを抽出します。
2.  **機能の特定**: 拡張機能の主要なセールスポイントを3〜5個特定します。
3.  **英語版のドラフト作成**: 上記の構造に従って英語版の説明文を作成します。
4.  **日本語版のドラフト作成**: 日本語版の説明文を作成します。単なる直訳ではなく、日本のユーザー向けにローカライズされていることを確認してください。
5.  **レビューと更新**: 既存の `store_description.md` と比較し、必要に応じて部分的な更新または全面的な上書きを行います。

## 例

### スキルの起動
「新しく追加したキーボードショートカットを含めてストアの説明文を更新して」
「現在の manifest を基に、日本語版のストア説明文を生成して」
