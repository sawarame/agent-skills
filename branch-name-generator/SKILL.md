# ブランチ名生成スキル

ユーザーから入力されれた要件に基づいて、Gitのブランチ名を生成するスキルです。ブランチ名はプロジェクトの規約や目的に応じて適切な命名規則を使用して生成されます。以下のガイドラインに従って、ブランチ名を作成してください。


## 1. 基本的な命名規則（ルール）
-   **すべて小文字、単語間はハイフン（`-`）で繋ぐ（ケバブケース）**
    -   `feat/add-login-function` 〇
    -   `feat/AddLoginFunction` ✕（大文字小文字の区別でトラブルが起きやすい）
    -   `feat/add_login_function` ✕（スネークケースはURLやCLIで少し扱いづらい場合がある）
-   **英数字とハイフン、スラッシュ（`/`）のみを使用する**
    -   日本語（全角文字）やスペース、特殊文字（`?`, `*`, `.` など）は、Gitのコマンドや外部ツールで予期せぬエラーの原因になるため避けます。

## 2. 最も一般的な構成（プレフィックスの活用）
ブランチの目的に応じて「プレフィックス（接頭辞）/ 概要」という形式にするのが現在の主流です。スラッシュを入れることで、SourceTree などのGUIツールやCLIでグループ分け（フォルダ分け）されて見やすくなります。

| プレフィックス | 用途 | 具体的ケース |
| --- | --- | --- |
| `feat/` | 新機能の開発 | `feat/user-profile-page` |
| `fix/` | バグの修正 | `fix/payment-button-error` |
| `hotfix/` | 本番環境の緊急バグ修正 | `hotfix/session-leak` |
| `docs/` | ドキュメントの作成・修正（READMEなど） | `docs/api-specification` |
| `style/` | コードの意味に変更を加えない修正（フォーマット、空白など） | `style/eslint-formatting` |
| `refactor/` | リファクタリング（機能追加やバグ修正を含まないコード整理） | `refactor/optimize-db-query` |
| `chore/` | ビルドプロセスやツール、ライブラリのアップデートなど雑多な作業 | `chore/update-dependencies` |

## 手順
1. ユーザーからブランチの目的や内容を聞き取ります。
2. 上記の命名規則とプレフィックスのガイドラインに従って、適切なブランチ名を生成してください。
3. 生成したブランチ名をユーザーに提示し、必要に応じて修正の提案を行ってください。
4. 最終的なブランチ名が決定したら、`git checkout -b 生成したブランチ名` を実行するかどうか、ユーザーに(y/n)で確認してください。
